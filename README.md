# ⭐ StarJobs — Telegram-бот поиска работы за звёзды

Telegram-бот, который **за звёзды в день** каждый день ищет вакансии по всей России и проверяет каждую через ИИ (OpenRouter), присылая только релевантные.

- Пользователь описывает желаемую работу и выбирает фильтры (должность, город, зарплата, опыт, формат, свободные пожелания).
- Бот ищет во всех подключённых источниках, проверяет соответствие через ИИ и присылает подборку.
- Оплата — нативными **Telegram Stars** (авто-продлеваемая подписка на 30 дней).

## Тарифы

| Тариф | Цена | ≈ в день | Проверка ИИ |
|-------|------|----------|-------------|
| **Базовый** | 30⭐ / 30 дней | ≈ 1⭐ | Бесплатные модели OpenRouter |
| **Про** | 3000⭐ / 30 дней | ≈ 100⭐ | `qwen3-32b` |

> **Почему подписка на 30 дней, а не «1⭐ в день»?** Telegram Stars нельзя
> списывать произвольно каждый день: нативная подписка имеет **фиксированный
> период 2 592 000 сек (30 дней)**, а списаниями и авто-продлением управляет
> сам Telegram. Поэтому «1⭐/день» = **30⭐/мес**, «100⭐/день» = **3000⭐/мес**.
> Потолок цены подписки — 10 000⭐ (Bot API 9.0), так что Про укладывается.
> Цены настраиваются через `BASIC_PRICE_STARS` / `PRO_PRICE_STARS`.

## Источники вакансий

- **HeadHunter** — `api.hh.ru`, без ключа, полный набор фильтров. Основной источник.
- **Работа России (Trudvsem)** — `opendata.trudvsem.ru`, открытые госданные, best-effort.

Добавить новый источник (SuperJob и т.п.) — реализовать интерфейс `JobSource`
из [`src/sources/base.ts`](src/sources/base.ts) и включить его в `SOURCES`
в [`src/sources/index.ts`](src/sources/index.ts).

## Как работает ИИ-проверка

Ключевой поиск в источниках даёт много «шумных» результатов. Поэтому каждую
вакансию-кандидата бот прогоняет через LLM: модель получает запрос кандидата и
описание вакансии и возвращает `{relevant, score 0-100, reason}`. В выдачу
попадают только вакансии со `score ≥ SCORE_THRESHOLD`.

- **Базовый** тариф → перебор бесплатных моделей (`OPENROUTER_FREE_MODELS`) с фолбэком.
- **Про** тариф → `qwen/qwen3-32b` (`OPENROUTER_PRO_MODEL`).
- **Нет ключа OpenRouter** → фолбэк на эвристику (совпадение по ключевым словам), бот остаётся рабочим.

## Быстрый старт

```bash
npm install
cp .env.example .env      # впиши BOT_TOKEN (и, по желанию, OPENROUTER_API_KEY)
npm run dev               # запуск с автоперезагрузкой (long polling)
```

Проверить поиск + ИИ без Telegram (не нужен BOT_TOKEN):

```bash
npm run try:search -- "python backend разработчик"
```

Прод-сборка:

```bash
npm run build && npm start
```

## Переменные окружения

Все — в [`.env.example`](.env.example). Ключевые:

| Переменная | Назначение |
|------------|------------|
| `BOT_TOKEN` | Токен от [@BotFather](https://t.me/BotFather) (обязательно) |
| `OPENROUTER_API_KEY` | Ключ [openrouter.ai/keys](https://openrouter.ai/keys). Пусто → эвристика |
| `OPENROUTER_FREE_MODELS` | Список бесплатных моделей через запятую |
| `OPENROUTER_PRO_MODEL` | Модель для Про (`qwen/qwen3-32b`) |
| `BASIC_PRICE_STARS` / `PRO_PRICE_STARS` | Цены тарифов в звёздах |
| `CRON_EXPR` / `CRON_TZ` | Расписание дневной рассылки (по умолчанию 09:00 МСК) |
| `SCORE_THRESHOLD` | Порог релевантности 0..100 |

> Бесплатные модели OpenRouter периодически меняются. Если модель из списка
> перестала быть бесплатной или недоступна — бот переберёт следующие, а затем
> эвристику. Актуальный список см. на [openrouter.ai/models](https://openrouter.ai/models) (фильтр *Free*).

## Команды бота

- `/start` — приветствие и меню
- `/search` — визард настройки поиска (6 шагов) + бесплатный превью-подбор
- `/status` — текущий поиск и статус подписки
- `/plan` — тарифы и оформление подписки
- `/stop` — отмена авто-продления
- `/help` — как всё работает

## Архитектура

```
src/
├── index.ts            # точка входа: long polling + крон дневной рассылки
├── bot.ts              # makeBot(): сборка и регистрация обработчиков
├── config.ts           # конфиг из env, тарифы, период подписки
├── types.ts            # доменные типы
├── store.ts            # персистентное JSON-хранилище (data/store.json)
├── format.ts / ui.ts   # HTML-карточки и inline-клавиатуры
├── sources/            # источники вакансий (hh, trudvsem) + агрегатор
├── ai/                 # OpenRouter-клиент и ИИ-проверка (+ эвристика)
├── jobs/               # pipeline (поиск→ИИ→отбор) и дневная рассылка
└── handlers/           # start, search (визард), subscription, payments
```

Поток оплаты (Telegram Stars, нативная подписка):

1. `createInvoiceLink` с `subscription_period=2592000`, `currency="XTR"` → ссылка-инвойс.
2. `pre_checkout_query` → мгновенное подтверждение.
3. `successful_payment` → активируем подписку, сохраняем `telegram_payment_charge_id`.
4. Telegram сам продлевает каждые 30 дней (новый `successful_payment`, `is_recurring=true`).
5. `/stop` → `editUserStarSubscription(is_canceled=true)` — сервис работает до конца оплаченного периода.

Крон **не списывает звёзды** — он только доставляет вакансии активным подписчикам
(биллинг полностью на стороне Telegram).

## Важно: HeadHunter и география сервера

`api.hh.ru` **гео-блокирует зарубежные серверные (дата-центровые) IP** и
отвечает `403 {"errors":[{"type":"forbidden"}]}` независимо от User-Agent.

- ✅ Деплой на **российском** хостинге (или с RU IP) — HH работает штатно.
- 🌍 Деплой **вне РФ** — заверни запросы через свой RU-прокси и укажи его в
  `HH_API_BASE` (например, лёгкий reverse-proxy на RU-VPS, проксирующий на `api.hh.ru`).
- 🛡 Бот в любом случае не падает: если HH недоступен, выдача формируется из
  остальных источников (`Promise.allSettled`), а Работа России доступна отовсюду.

## Ограничения MVP / что дальше

- Хранилище — JSON-файл (`DATA_DIR`). Для масштаба заменить `src/store.ts` на Redis/Postgres.
- Один активный поисковый запрос на пользователя (легко расширить до нескольких).
- Long polling — для прод-вебхука подключить `bot.api.setWebhook` и HTTP-сервер.
- Avito / Habr Career не подключены (нет публичного API).
