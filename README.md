# 🤖 AI Career Agent

Личный Telegram-агент поиска работы. Каждый день ищет вакансии по **двум трекам**,
оценивает каждую через ИИ (OpenRouter) и постит подборку в **топики** одной супергруппы.
Личный инструмент для одного владельца — без биллинга и подписок.

> Форк движка StarJobs, переписанный под персонального агента. Полное видение —
> [`docs/AI_Career_Agent_Spec_v2.md`](docs/AI_Career_Agent_Spec_v2.md), план работ —
> [`docs/IMPLEMENTATION_PLAN.md`](docs/IMPLEMENTATION_PLAN.md).

## Два трека

- **Track A — Руководитель проектов (продакшн):** прямое совпадение с текущим опытом.
- **Track B — Product / Project Manager (IT):** переход через transferable skills
  (переформулировка продакшн-опыта на язык IT-менеджмента).

Каждый трек — отдельный набор фильтров, весов скоринга и топик в группе.
Конфиг треков — в [`src/tracks/index.ts`](src/tracks/index.ts).

## Как устроен UI: супергруппа с топиками

Один бот постит в одну супергруппу с включёнными **Темами (Forum Topics)**:

- топик трека A, топик трека B — карточки вакансий с кнопками действий;
- топик «Отклики» — черновики сопроводительных писем (в разработке);
- топик «Аналитика» — еженедельный дайджест (в разработке).

Привязка топиков — командой `/bind` **внутри** нужного топика (бот сам берёт
`chat_id` и `message_thread_id`).

## Быстрый старт (локально)

```bash
npm install
cp .env.example .env      # впиши BOT_TOKEN и OWNER_ID
npm run build && npm start
```

Проверить поиск + скоринг без Telegram:

```bash
npm run try:search -- A          # трек A
npm run try:search -- "product manager"
```

## Команды бота

- `/bind A|B|inbox|digest` — привязать текущий топик
- `/run [A|B]` — искать сейчас (без аргумента — оба трека)
- `/status` — привязки топиков и воронка статусов
- `/help` — справка

## Переменные окружения

Все — в [`.env.example`](.env.example). Ключевые:

| Переменная | Назначение |
|------------|------------|
| `BOT_TOKEN` | Токен от [@BotFather](https://t.me/BotFather) (обязательно) |
| `OWNER_ID` | Твой Telegram id — единственный, кому подчиняется бот |
| `OPENROUTER_API_KEY` | Ключ [openrouter.ai/keys](https://openrouter.ai/keys). Пусто → эвристика |
| `OPENROUTER_SCORE_MODELS` | Модели скоринга (через запятую, с фолбэком) |
| `LETTER_MODEL` | Сильная модель для сопроводительных писем |
| `HH_API_BASE` | База HH; RU-прокси при деплое вне РФ |
| `HH_USER_AGENT` | **Реальный** контакт — плейсхолдеры HH банит |
| `CRON_EXPR` / `CRON_TZ` | Расписание ежедневного поиска |
| `SCORE_THRESHOLD` | Порог релевантности 0..100 |

## ⚠️ HeadHunter и география сервера

`api.hh.ru` **гео-блокирует зарубежные дата-центровые IP** — отвечает
`403 forbidden` независимо от User-Agent. Кроме того, HH **банит плейсхолдерные
User-Agent** (`example.com`, `test`) с ошибкой `400 bad_user_agent`.

- ✅ Деплой на **российском** IP — HH работает штатно (с реальным `HH_USER_AGENT`).
- 🌍 Деплой **вне РФ** (Fly.io и т.п.) — заверни запросы через RU-прокси и укажи
  его в `HH_API_BASE`. Иначе трек по HH не работает.
- 🛡 Бот не падает: если HH недоступен, выдача формируется из остальных источников
  (`Promise.allSettled`).

## Статус реализации

Готово (Фаза 0 — каркас): личный режим (owner-guard), два трека, привязка топиков
`/bind`, постинг карточек в топики с кнопками действий, дедуп, JSON-хранилище.

В работе (см. [`docs/IMPLEMENTATION_PLAN.md`](docs/IMPLEMENTATION_PLAN.md)):
полный AI-скоринг с весами по треку, богатые карточки, генерация сопроводительных
писем, полуавто-отклик, статусы-воронка, еженедельный дайджест, деплой-инфра Fly.io.

## Архитектура

```
src/
├── index.ts          # точка входа: long polling + крон
├── bot.ts            # makeBot(): owner-guard + регистрация хендлеров
├── config.ts         # конфиг из env
├── types.ts          # доменные типы (треки, вакансии, статусы)
├── store.ts          # JSON-хранилище (топики, дедуп, вакансии)
├── tracks/           # конфиги треков A и B (фильтры, веса, резюме)
├── format.ts / ui.ts # карточки и inline-кнопки
├── sources/          # источники вакансий (hh, trudvsem) + агрегатор
├── ai/               # OpenRouter-клиент и ИИ-скоринг (+ эвристика)
├── jobs/             # pipeline (поиск→скоринг→отбор) и прогон по трекам
└── handlers/         # commands (/bind, /run, /status), actions (кнопки)
```
